# OpenFuncDB 数据处理流程

## 数据来源

当前使用 Python 官方文档作为首批数据源：

- **Python 内置函数**: https://docs.python.org/3/library/functions.html
- 原始数据文件: `data/source/python_official_raw.json`
- 数据量: 68 条 Python 内置函数

## 处理流程

### 流程概览

```
官方源 → 原始数据落盘 → 三层清洗 → AI 标准化 → JSON 输出 → 事务入库 → 数据校验
```

### 第一步：官方源获取

解析官方 HTML 文档，提取函数名称和描述信息。相比爬虫更稳定、不易被封。

### 第二步：原始数据落盘

原始 JSON 格式：

```json
[
  {
    "func_name": "abs",
    "func_desc": "返回一个数字的绝对值..."
  }
]
```

### 第三步：三层清洗

1. **去噪清洗**：去除空白字符、格式乱码、HTML 标签残留
2. **规则清洗**：过滤非函数条目、过长文本、特殊字符
3. **质量清洗**：去重、补缺失字段、统一格式

### 第四步：AI 标准化

将原始自然语言描述映射到数据库字段：

| 原始字段 | 目标字段 | 说明 |
|---------|---------|------|
| func_name | func_base.func_name | 函数名 |
| func_desc | func_base.func_desc | 中文描述 |
| AI 结构化 | func_base.func_content | 代码签名 |
| AI 结构化 | func_base.func_params | 参数列表 |
| AI 结构化 | func_base.func_return | 返回值类型 |
| 规则判定 | func_base.func_type | 函数类型（固定为 Python） |

### 第五步：输出标准 JSON

统一格式输出：

```json
{
  "func_type": "Python",
  "func_name": "abs",
  "func_content": "abs(x)",
  "func_desc": "返回数字的绝对值",
  "func_params": "x",
  "func_return": "number",
  "is_safe": 1
}
```

### 第六步：事务化入库

使用 SQLAlchemy ORM 进行事务性批量写入：

- 开启事务
- 不存在则插入，存在则更新（去重）
- 同时写入 `func_base` 和 `func_category_relation`
- 输出入库报告

### 第七步：数据校验

入库后自动校验：
- 字段非空校验
- 函数名唯一性校验
- 安全等级校验
