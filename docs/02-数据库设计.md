# OpenFuncDB 数据库设计

## 版本记录

| 版本 | 日期 | 说明 |
|------|------|------|
| v2.0 | 2026-05 | 重构为 SQLAlchemy ORM 模型，替换旧版自增建表 |

## 核心表结构

### func_base — 函数/命令主表

存储所有函数和命令的核心数据。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| func_type | VARCHAR(50) | NOT NULL, INDEX | 类型（Python/Linux/Git/Shell） |
| func_name | VARCHAR(200) | NOT NULL | 函数/命令名称 |
| func_content | TEXT | NOT NULL | 代码内容/使用示例 |
| func_desc | TEXT | | 功能描述 |
| func_params | TEXT | | 参数说明 |
| func_return | TEXT | | 返回值说明 |
| is_safe | TINYINT | DEFAULT 0 | 安全标志（0=待审核, 1=安全） |
| create_time | DATETIME | DEFAULT NOW | 创建时间 |
| update_time | DATETIME | ON UPDATE NOW | 更新时间 |

### func_category — 分类表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| category_name | VARCHAR(100) | NOT NULL | 分类名称 |
| func_type | VARCHAR(50) | INDEX | 关联类型 |
| category_desc | TEXT | | 分类描述 |

### func_category_relation — 分类关联表

多对多关联函数与分类。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| func_id | INT | FK -> func_base.id | 函数 ID |
| category_id | INT | FK -> func_category.id | 分类 ID |

### func_audit — 函数审核表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| func_id | INT | FK -> func_base.id | 函数 ID |
| audit_status | INT | DEFAULT 0 | 状态（0=待审核, 1=通过, 2=拒绝） |
| audit_user | VARCHAR(100) | | 审核人 |
| audit_time | DATETIME | | 审核时间 |
| audit_remark | TEXT | | 审核备注 |

### users — 用户鉴权表

替代旧版的 `api_auth` 表。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(100) | UNIQUE, NOT NULL, INDEX | 用户名 |
| hashed_password | VARCHAR(255) | NOT NULL | 加密密码 |
| auth_level | VARCHAR(50) | DEFAULT 'user' | 权限级别（user/admin/super_admin） |
| is_active | TINYINT | DEFAULT 1 | 是否激活 |
| create_time | DATETIME | DEFAULT NOW | 创建时间 |

## ER 关系

```
func_base 1 ── * func_category_relation * ── 1 func_category
func_base 1 ── * func_audit
```

## 变更记录

旧版（已弃用）：
- `api_auth` 表 → 替换为 `users` 表
- `api_call_log` 表 → 已移除（日志功能后续重建）

## 数据库迁移

使用 Alembic 管理数据库版本：

```bash
# 生成迁移脚本
python -m alembic revision --autogenerate -m "描述"

# 升级到最新版本
python -m alembic upgrade head

# 回滚一个版本
python -m alembic downgrade -1
```
